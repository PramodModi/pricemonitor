## [v1.0] — MVP — July 2026

### Features

#### FEAT-001 — Subscription Confirmation Email
- **Affects:** PRD §7, Email Template Design Spec
- **Change:** New email sent on successful subscription with product details and affiliated product URL
- **Implementation:** `EmailSender.send_subscription_confirmation()` — `app/notifications/email_sender.py`
- **Content:** `CONFIRMATION_SUBJECT` — `app/notifications/content/price_drop.py`

#### FEAT-002 — Affiliate Tag in Stored URLs
- **Affects:** SAD §8.3, LLD §8.3
- **Change:** Affiliate tag appended to `products.url` at subscription time in `ProductSyncService.sync()`
- **Implementation:** `app/services/product_sync.py`
- **Config:** `AMAZON_AFFILIATE_TAG`, `FLIPKART_AFFILIATE_ID` in `.env` and Railway Variables

#### FEAT-003 — Amazon URL Reconstruction
- **Affects:** LLD §10
- **Change:** `URLValidator` now reconstructs clean `https://www.amazon.in/dp/{ASIN}` URL from any Amazon URL format — cart links, image links, `/gp/product/`, search result URLs
- **Implementation:** `URLValidator._reconstruct_amazon_url()` — `app/services/url_validator.py`

#### FEAT-004 — Price Change Updates current_price in Both Directions
- **Affects:** LLD §12.1, SAD §9.5
- **Change:** `products.current_price` now updated when price changes in either direction (up or down). Email notifications still only sent on price drops.
- **Implementation:** `ScraperWorker._write_result()` — `app/workers/scraper_worker.py`

### Deviations from Design

#### DEV-001 — ProductSnapshot.live_data stored as dict
- **Affects:** LLD §9
- **Original:** `live_data` typed as `LiveData` Pydantic object
- **Actual:** Stored as plain `dict`
- **Reason:** Avoids circular imports between service layer and schema layer

#### DEV-002 — HTML built directly in Python for Streamlit components
- **Affects:** Streamlit UI Design §9
- **Original:** HTML loaded from `static/html/` template files
- **Actual:** HTML built directly as Python f-strings in `preview_card.py` and `product_card.py`
- **Reason:** Python `.format()` misinterprets CSS class attributes (e.g. `class="product-meta"`) as format variables. `str.replace()` with individual calls used instead.

#### DEV-003 — Streamlit imports use sys.path instead of package prefix
- **Affects:** Streamlit UI Design §11
- **Original:** `from streamlit_app.x import y`
- **Actual:** `sys.path.insert(0, ...)` at top of each file, then `from x import y`
- **Reason:** Streamlit Cloud runs files from a different working directory — package prefix causes `ModuleNotFoundError`

#### DEV-004 — playwright_stealth API changed
- **Affects:** LLD §11, §12
- **Original:** `stealth_sync(page)`
- **Actual:** `Stealth().apply_stealth_sync(page)`
- **Reason:** playwright_stealth v2.0.3 changed the API — `stealth_sync` removed, replaced with `Stealth` class

#### DEV-005 — ORM models split into separate files
- **Affects:** LLD §5
- **Original:** All models in single `app/core/models.py`
- **Actual:** Each model in separate file under `app/core/models/`
- **Reason:** Cleaner separation, easier to test individually

#### DEV-006 — Logger calls use f-strings throughout
- **Affects:** LLD §17
- **Original:** Structured keyword arguments — `logger.info("msg", key=value)`
- **Actual:** f-string format — `logger.info(f"msg — key={value}")`
- **Reason:** Standard Python `logging.Logger` does not accept keyword arguments

### Deployment Notes
